## 经验卡片 #runtime-001：模板字面量内嵌 JS 需单独语法验证

- **触发场景：** VS Code 插件 WebView 面板中，HTML 通过模板字面量生成，内嵌的 `<script>` 标签包含大量 JS 逻辑
- **关联元动作：** [M5 转换]（模板字面量转义）、[F12 捕获]（语法错误检测）、[I13 渲染]
- **问题描述：** `edit_file` 在模板字面量内对 `\\` 双重转义导致嵌入的 JS 正则表达式语法全坏。但 `node --check probe-panel.js` 只验证 Node.js 外层语法，不验证模板字面量输出的 HTML 中 `<script>` 的内容。导致语法错误在开发阶段完全漏过，运行时 WebView JS 静默崩溃，用户看到「没反应」
- **根本原因：** [M5 转换] 验证范围不足——`node --check` 只覆盖 Node.js 层，未覆盖模板字面量动态生成的代码
- **解决方案：** 
  1. 每次修改含嵌入 JS 的模板字面量后，提取 `<script>` 内容单独跑 `node --check`
  2. 在模板字面量内写正则时，避免使用 `\\` 转义——改用「先 escapeHtml 转义全文，再对已转义文本做简单正则」的策略
- **预防规则：** 任何包含模板字面量内嵌 JS 的文件，修改后必须执行验证脚本：提取 `<script>` 内容 → `node --check`
- **工具链映射：** 开发流程 — 修改含嵌入 JS 的模板字面量后必须验证
